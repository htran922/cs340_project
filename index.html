Database Outline: Smithsonian National Museum of Natural History

The entities in our database are:

  • Staff

    This entity represents museum staff

    • *Staff ID

      [Data Type: Integer] - This number is automatically assigned to each staff member when they are recorded in our database. An auto-incrementing number which is the primary key.

    • First Name

      [Data Type: varchar] - First name of the staff member which is a string of maximum 50 characters. It cannot be blank and there is no default.

    • Last Name

      [Data Type: varchar] - Last name of the staff member which is a string of maximum 50 characters. It cannot be blank and there is no default. Job/Role [Data Type: varchar] - Staff member's position which is a string of maximum 100 characters. It cannot be blank and there is no default.

  • Artifact

    This entity represents the items in the museum's collection.

    • *Artifact ID

      [Data Type: Integer] - This number is automatically assigned to each artifact when they are recorded in our database. An auto-incrementing number which is the primary key.

    • Display

      [Data Type: Boolean] - If True the artifact is currently on display, if False the artifact is in storage.

    • Acquisition Year

      [Data Type: Integer] - Year the artifact was added to the museum's collection, which is an integer of maximum 3000.

    • Material

      [Data Type: varchar] - What the artifact is made out of, which is a string of maximum 100 characters.

    • Damage

      [Data Type: Boolean] - If True the artifact has sustained some sort of damage. If False the artifact does not have known damage.

    • Photo

      [Data Type: varchar] - Name of the image file containing a photograph of the artifact, which is a string of maximum 100 characters.

  • Exhibition

    This entity represents the showcases planned by curators to display museum artifacts and teach the public.

    • *Exhibition ID

      [Data Type: Integer] - This number is automatically assigned to each exhibition when they are recorded in our database. An auto-incrementing number which is the primary key.

    • *Artifact ID

      [Foreign Key, Data Type: Integer] - ID of artifacts featured in the exhibition.

    • Room

      [Data Type: varchar] - Name of the room where the exhibition is held, which is a string of maximum 100 characters.

    • Subject

      [Data Type: varchar] - Academic topic covered by exhibition, which is a string of maximum 100 characters. For example: Geology, Earth Science, Insectology.

    • Start Year

      [Data Type: Integer] - Start year of the exhibition, which is an integer of maximum 3000.

    • Start Month

      [Data Type: Integer] - Start month of the exhibition, which is an integer of maximum 12.

    • Start Day

      [Data Type: Integer] - Start day of the exhibition, which is an integer of maximum 31.

    • End Year

      [Data Type: Integer] - End year of the exhibition, which is an integer of maximum 3000.

    • End Month

      [Data Type: Integer] - End month of the exhibition, which is an integer of maximum 12.

    • End Day

      [Data Type: Integer] - End day of the exhibition, which is an integer of maximum 31.

  • Event

    This entity represents the events held at various exhibitions at the museum.

    • *Event ID

      [Data Type: Integer] - This number is automatically assigned to each event when they are recorded in our database. An auto-incrementing number which is the primary key.

    • *Exhibition ID

      [Foreign Key, Data Type: Integer] - ID of the exhibition with which the event is associated.

    • *Staff ID

      [Foreign Key, Data Type: Integer] - IDs of the staff members who worked on an event.

    • Type of Event

      [Data Type: varchar] - Name of the type of event that is occuring, which is a string of maximum 100 characters. Includes concerts, lectures, and galas.

    • Year

      [Data Type: Integer] - Year the event took place, which is an integer of maximum 3000.

    • Month

      [Data Type: Integer] - Month the event took place, which is an integer of maximum 12.

    • Day

      [Data Type: Integer] - Day the event took place, which is an integer of maximum 31.

  • Artifact_Exhibition

    This associative entity connects the many-to-many relationship of Artifacts and Exhibitions.

    • Artifact ID

      [Foreign Key, Data Type: Integer] - ID of artifacts featured in the exhibition.

    • Exhibition ID

      [Foreign Key, Data Type: Integer] - This number is automatically assigned to each exhibition when they are recorded in our database.

  • Staff_Exhibition

    This associative entity connects the many-to-many relationship of Staff and Exhibitions.

    • Staff ID

      [Foreign Key, Data Type: Integer] - IDs of the staff members who worked on an event.

    • Exhibition ID

      [Foreign Key, Data Type: Integer] - ID of the exhibition with which the event is associated.

  • Staff_Event

    This associative entity connects the many-to-many relationship of Staff and Events.

    • Staff ID

      [Foreign Key, Data Type: Integer] - This number is automatically assigned to each staff member when they are recorded in our database.

    • Event ID

      [Foreign Key, Data Type: Integer] - This number is automatically assigned to each event when they are recorded in our database.
